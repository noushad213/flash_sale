
# Product Vision

Build a **Flash Sale Commerce Platform** capable of handling:

* extreme concurrency
* limited inventory
* real-time traffic bursts
* strict purchase correctness

The system should feel like a **real commerce infrastructure platform**, not just a UI.

Think:

**Shopify Flash Sale + Stripe Infrastructure Dashboard**

---

# Core System Modules

The platform should contain **two major apps**:

### 1️⃣ Customer App (Buyer Side)

Where users attempt to buy the product.

### 2️⃣ Admin / System Dashboard

Where engineers monitor the flash sale.

---

# App Architecture

```
Frontend (Next.js / React)

    ├── Customer App
    └── Admin Dashboard

Backend (Node / FastAPI)

    ├── Checkout API
    ├── Inventory Service
    ├── Order Service
    ├── Payment Simulation
    └── Queue Processor

Infrastructure

    ├── Redis (concurrency gate)
    ├── Queue (BullMQ / Kafka style)
    ├── Database (Postgres)
    └── WebSockets (live updates)
```

## Purchase Flow

```
User Clicks Buy
      ↓
Request Sent
      ↓
Queue Position Assigned
      ↓
Processing
      ↓
Success / Failed
```

Show **queue position**.

Example:

```
You are #342 in line
Processing purchase request...
```

This demonstrates **traffic handling**.

# Virtual Waiting Room

This is **a major flash-sale feature** used by big companies.

When traffic spikes:

```
Users enter waiting room
Requests are gradually processed
```

UI example:

```
Flash Sale in Progress

You are in the queue.

Position: 218
Estimated Time: 12 seconds
```
